1. Purpose & Task

The purpose of this laboratory work is to study basic arithmetic and conversion operators in a programming language and to compare the chosen language with other programming languages known by the student. Ruby was selected for implementation because it has concise syntax, expressive built-in numeric operations, and readable string interpolation.

Tasks completed in this report:

  1. Read two integers and output their product and integer quotient.
  2. Read three real numbers, compute their sum, and output the result with two decimal places.
  3. Calculate the volume of a room in cubic metres and cubic centimetres.
  4. Ask the user for age in years and output the age on the next birthday.
  5. Express a number of days in weeks and days.
  6. Express a number of seconds in hours, minutes, and seconds.
  7. Convert kilometres per hour to miles per hour using 1 km = 0.62137 mile.

2. Program Code & Results

2.1. Task 1

Write a program which reads two integers and displays their product and the integer quotient when the first is divided by the second.

Program Code:

#!/usr/bin/env ruby

def read_integer(prompt)
  loop do
    print prompt
    input = gets&.strip
    exit if input.nil?

    value = Integer(input, exception: false)
    return value unless value.nil?

    puts 'Please enter a valid integer.'
  end
end

first_number = read_integer('Enter the first integer: ')
second_number = read_integer('Enter the second integer: ')

product = first_number * second_number

puts "Product: #{product}"

if second_number.zero?
  puts 'Integer quotient: undefined (division by zero).'
else
  quotient = first_number.div(second_number)
  puts "Integer quotient (DIV): #{quotient}"
end

2.2. Task 2

Write a program which accepts three real numbers, computes their sum, and outputs the result. Amend the program so that the result is output to 2 decimal places.

def read_float(prompt)
  loop do
    print prompt
    input = gets&.strip

end

first_number = read_float('Enter the first real number: ')
second_number = read_float('Enter the second real number: ')
third_number = read_float('Enter the third real number: ')

sum = first_number + second_number + third_number

puts "Sum: #{sum}"
puts format('Sum rounded to 2 decimal places: %.2f', sum)

2.3. Task 3

Write a program that takes in the width, length, and height of a room in metres and returns the volume of the room in cubic metres and cubic centimetres.

Program Code:

#!/usr/bin/env ruby

def read_float(prompt)
  loop do
    print prompt
    input = gets&.strip
    exit if input.nil?

    value = Float(input, exception: false)
    if !value.nil? && value >= 0
      return value
    end

    puts 'Please enter a valid non-negative number.'
  end
end

width = read_float('Enter the room width in metres: ')
length = read_float('Enter the room length in metres: ')
height = read_float('Enter the room height in metres: ')

volume_cubic_metres = width * length * height
volume_cubic_centimetres = volume_cubic_metres * 1_000_000

puts format('Volume: %.3f cubic metres', volume_cubic_metres)
puts format('Volume: %.0f cubic centimetres', volume_cubic_centimetres)

3. Conclusions

All seven tasks were successfully implemented in Ruby. The programs use the basic arithmetic operations of the language, integer division with div, quotient and remainder extraction with divmod, floating-point calculations, and formatted output through format. Sample runs confirm that the expected results were achieved.

Ruby is very convenient for this kind of assignment because its syntax is concise and easy to read. Compared with C++ or Java, Ruby requires much less boilerplate code: no explicit type declarations, no semicolons, and no class wrapper for simple console programs. Compared with Python, Ruby is similarly readable, but it provides some expressive numeric helpers such as div and divmod that map naturally to the task requirements.

The main advantage of Ruby in this laboratory work is readability and speed of development. The main disadvantage is that Ruby is dynamically typed, so some mistakes are detected only at runtime rather than at compile time. For short educational programs, however, Ruby is a strong choice because it lets the programmer focus on the task logic instead of language ceremony.
